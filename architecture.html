The Stack

A layered
runtime for
autonomy.

/// OVERVIEW

Dolphin's architecture is composed of seven layers that turn a model into an agent — and an agent into a reliable system.

Each layer is independently observable, swappable, and policy-bounded. Together they form the substrate of autonomous behavior.

/// DOLPHIN_STACK · v0.7.2 read top-down
L7 Goal & intent objective decomposition · constraint capture
L6 Planning multi-step plans · branch evaluation
L5 Memory episodic · semantic · working state
L4 Reasoning core model routing · cost/quality optimization
L3 Tool runtime sandbox · timeouts · structured I/O
L2 Perception vision · audio · documents · live feeds
L1 Environment software · hardware · network surface
03 · Principles

Design principles.

Five non-negotiables that shape every layer of the system.

/// 01

Composable, not monolithic

Each subsystem — memory, planning, tools, perception — has a clean interface and can be replaced without rewriting the agent.

LAYERED→
/// 02

Observable by default

Every reasoning step, tool call, and state transition is logged with a structured trace. Replay is a first-class operation.

TRACED→
/// 03

Bounded by policy

Capabilities are declared and gated. Agents cannot exceed permissions; sensitive actions require attestation.

GATED→
/// 04

Learn from outcomes

Every task closes a feedback loop. Evaluation signals flow back into routing, prompts, and policy weights.

ADAPTIVE→
/// 05

Real-world first

The architecture is designed for messy environments — partial information, flaky APIs, hardware in the loop, long horizons.

GROUNDED→
/// 06

Model-agnostic

Route between open-weight, frontier, and specialist models at the reasoning layer. No vendor lock-in.

NEUTRAL→
04 · Loop

The decision loop.

An agent in Dolphin is not a chatbot. It's a closed-loop process running continuously, with every iteration logged and verifiable.

/// CONTROL_LOOP cycle ≈ 12ms
→Observeingest environment state
→Orientretrieve memory, ground intent
→Decideplan + select action
→Actinvoke tool · check policy
→Verifyevaluate outcome · update memory
↻Repeatuntil goal · or escalation
